# Estructura de la Arquitectura .NET Core - UrbanControl.Backend\n\nEsta es una **aplicación Web API estándar de .NET Core/ASP.NET Core** con **Clean Architecture** básica, enfocada en un sistema de gestión de urbanizaciones (proyectos, lotes, clientes, reservas) con un sistema de **permisos RBAC (Role-Based Access Control)** sofisticado.\n\n## 🎯 **Propósito del Proyecto**\nSistema de control urbano para manejar:\n- **Proyectos** (urbanizaciones)\n- **Lotes** (terrenos dentro de proyectos) \n- **Clientes** y **Reservas**\n- **Sistema de Autenticación y Permisos Avanzado**\n\n## 🏗️ **Estructura General (Solution-Level)**\n```\nUrbanControl.Backend.sln          ← Solución Visual Studio\n├── Dockerfile                    ← Contenedor Docker\n├── README.md                     ← Documentación\n└── UrbanControl.Backend/         ← Proyecto principal (Web API)\n```\n\n## 📁 **Estructura del Proyecto Principal (.NET Core Web API)**\n\n```\nUrbanControl.Backend/\n├── Program.cs                    ← Punto de entrada (Minimal API/Startup)\n├── UrbanControl.Backend.csproj   ← Configuración del proyecto\n├── Properties/launchSettings.json\n├── Controllers/                  ← **REST API Controllers**\n│   ├── AuthController.cs         ← Autenticación/Login\n│   ├── ClienteController.cs      ← CRUD Clientes\n│   ├── LoteController.cs         ← CRUD Lotes\n│   ├── ProyectoController.cs     ← CRUD Proyectos\n│   ├── ReservaController.cs      ← CRUD Reservas\n│   ├── TestController.cs         ← Controller de pruebas\n│   └── access_privilegio/\n│       └── PermisosController.cs ← Gestión de permisos RBAC\n├── Models/                       ← **Entidades del Dominio**\n│   ├── Cliente.cs\n│   ├── Lote.cs\n│   ├── Proyecto.cs\n│   ├── Reserva.cs\n│   ├── User.cs\n│   └── access_privilegios/       ← **Modelo de Permisos**\n│       ├── Modulo.cs\n│       ├── Submodulo.cs\n│       ├── Rol.cs\n│       ├── PermisoRol.cs\n│       ├── CapacidadSubmodulo.cs\n│       └── TipoPermiso.cs\n├── DTOs/                         ← **Data Transfer Objects**\n│   ├── LoginDto.cs, UserDto.cs\n│   ├── ClienteDto.cs, LoteDto.cs\n│   ├── ProyectoDto.cs\n│   ├── ReservaDto.cs\n│   └── acessos_privilegios/      ← DTOs de permisos\n│       ├── PermisoRolUpdateDto.cs\n│       ├── PermisoMatrizDto.cs\n│       └── SubmoduloDto.cs\n├── Services/                     ← **Capa de Servicios (Business Logic)**\n│   ├── IAuthService.cs → AuthService.cs\n│   ├── IClienteService.cs → ClienteService.cs\n│   ├── ILoteService.cs → LoteService.cs\n│   ├── IProyectoService.cs → ProyectoService.cs\n│   ├── IReservaService.cs → ReservaService.cs\n│   └── IPermisosService.cs → PermisosService.cs\n├── Data/                         ← **Capa de Datos**\n│   └── ApplicationDbContext.cs   ← Entity Framework Core DbContext\n└── Migrations/                   ← **Migraciones EF Core** (6+ migraciones)\n```\n\n## 🔧 **Patrones Arquitectónicos Implementados**\n\n1. **Repository Pattern** (implícito via DbContext)\n2. **Service Layer Pattern** (Separación de lógica de negocio)\n3. **DTO Pattern** (Separación entre Models y API contracts)\n4. **RBAC Authorization** (Roles + Permisos por Módulo/Submódulo)\n5. **RESTful API Design**\n\n## 🗄️ **Base de Datos (Entity Framework Core)**\n```\nEntidades principales:\n├── Proyecto → Lote (1:N)\n├── Lote → Reserva (1:N) \n├── Cliente → Reserva (1:N)\n├── User → Rol (RBAC)\n└── Jerarquía Permisos:\n   Modulo → Submodulo → PermisoRol → TipoPermiso\n```\n\n## 🚀 **Flujo Típico de la Aplicación**\n```\nCliente → AuthController → AuthService → JWT Token\nCliente → ClienteController → ClienteService → DbContext\nAdmin → PermisosController → PermisosService → Gestión RBAC\n```\n\n## ✅ **Estado del Proyecto**\n- **Proyecto maduro** con múltiples migraciones EF Core\n- **Sistema de permisos completamente implementado**\n- **CRUD completo** para todas las entidades principales\n- **Listo para Docker** (`Dockerfile` presente)\n- **API completamente expuesta** via Controllers REST\n\n## 🎨 **Arquitectura Resumida**\n```\n┌─────────────────┐    ┌──────────────────┐    ┌─────────────┐\n│   Controllers   │───▶│    Services      │───▶│  DbContext  │\n│  (REST API)     │    │ (Business Logic) │    │ (EF Core)   │\n└─────────────────┘    └──────────────────┘    └─────────────┘\n         │                       │                       │\n    ┌──────────────┐       ┌──────────────┐      ┌─────────────┐\n    │     DTOs     │       │    Models     │      │  Database   │\n    │ (Contracts)  │       │ (Entities)    │      │ (SQL Server│\n    └──────────────┘       └──────────────┘      │  o SQLite)  │\n                                                 └─────────────┘\n```\n\n**Es una arquitectura .NET Core sólida, escalable y bien organizada para un sistema de gestión inmobiliaria/urbanística con control de accesos avanzado.**
